# Lang Master — 项目愿景与架构

> 从自然语言学与编程哲学的视角，帮助初学者精通编程语言排行榜 Top 10 语言，并掌握超越单一语言的元能力。

## 1. 灵感来源：Typing Master 的设计映射

Typing Master（多语言打字练习桌面应用）的核心设计模式，可以直接迁移到编程语言学习场景：

| Typing Master 概念 | Lang Master 对应概念 | 说明 |
|---|---|---|
| 6 种自然语言 | Top 10 编程语言 | 从 fr/es/it/pt/ru/ja → Python/C/Java/C++/C#/JS/Go/Rust/Swift/R |
| 词汇数据模块 (`words/*.js`) | 语言特性数据模块 (`langs/*.json`) | 每种语言的语法、语义、惯用法数据集 |
| 分类体系 (salutations/nourriture...) | 编程范式分类 (OOP/FP/并发/系统...) | 按概念域组织而非按语言 |
| 难度分级 (A1→C1 / N5→N1) | 编程难度分级 (P1→P5) | 从表达式→模块→并发→元编程 |
| 虚拟键盘多布局 (US Intl/ЙЦУКЕН/JIS) | 编程范式视图切换 (过程式/OOP/函数式) | 同一问题在不同范式下的表达方式 |
| 3 种游戏模式 (自由/限时/精确) | 3 种学习模式 (探索/挑战/精通) | 从宽到深的渐进练习 |
| 实时反馈 (逐字符校验) | 逐概念校验 | 不是字符级而是概念级即时验证 |
| 死键机制 (accent → 组合字符) | 概念组合 (primitive → compound concept) | 基本概念组合产生高级构造 |
| 手册生成器 (按语言动态生成) | 语言手册生成器 (按范式动态生成) | 从自然语言手册→编程语言手册 |
| 统计追踪 (WPM/准确率/错误字符) | 学习进度追踪 (概念掌握率/范式切换速度) | 量化元能力而非单一语言速度 |
| CSV 导入/导出 | 课程包导入/导出 | 可扩展的内容管理系统 |

**关键复用决策：** Electron 桌面端 + vanilla JS 架构保留，但核心逻辑从「打字速度训练」转向「概念理解训练」。

## 2. 核心创新：语言学-编程学双层映射

### 2.1 自然语言学五层结构 → 编程语言五层结构

| 语言学层次 | 定义 | 编程语言对应 | 示例 |
|---|---|---|---|
| **语音层 (Phonology)** | 声音的组织规律 | **词法层 (Lexical)** — token/关键字/运算符的合法形式 | `if`, `->`, `===`, `fn` |
| **形态层 (Morphology)** | 词的内部结构与派生 | **语法构造层 (Syntactic Morphology)** — 表达式/语句的内部组合规则 | `a + b`, `x.y()`, `&mut T` |
| **句法层 (Syntax)** | 词如何组合成句子 | **句法层 (Syntax)** — 语句/块/模块的组合规则 | `for..in`, `class X extends Y`, `match x { }` |
| **语义层 (Semantics)** | 句子的意义 | **语义层 (Semantics)** — 程序行为的指称 | 求值规则、类型系统、副作用语义 |
| **语用层 (Pragmatics)** | 语境中的使用 | **语用层 (Pragmatics)** — 设计模式/惯用法/生态系统惯例 | Pythonic, Go's `if err != nil`, Rust's `Result<T,E>` |

**教学法意义：** 初学者通常只学 Syntax（句法），但精通一门语言需要掌握全部五层。Lang Master 将五层结构显式化，让学习者知道自己处于哪一层、还需攻克哪一层。

### 2.2 Sapir-Whorf 假说的编程版

**弱形式（已被广泛接受）：** 编程语言影响（但不决定）程序员的思维模式。

| 语言 → 思维模式影响 | 典型表现 |
|---|---|
| Python → "一切皆对象，简洁优先" | 开发者倾向用最短代码解决问题，关注可读性 |
| C → "手动管理，精确控制" | 开发者对内存布局有直觉，追求零开销抽象 |
| Java → "一切皆类型，安全优先" | 开发者习惯用接口和继承组织复杂关系 |
| Rust → "所有权思维，零妥协安全" | 开发者自然思考资源生命周期，排斥隐式拷贝 |
| Haskell → "纯函数思维，数学优先" | 开发者排斥副作用，优先用组合而非变异 |
| Go → "简洁并发，实用主义" | 开发者用 goroutine 而非复杂抽象处理并发 |

**元能力目标：** 让学习者意识到不同语言的思维框架，并能主动切换——而非被单一语言锁定。

### 3.3 语言亲缘树（类比语言系属分类）

```
编程语系树（类比印欧语系 / 感谢语族分类）

Algol 语族 (≈ 印欧语系)
├── C 分支 (≈ 感谢语族日耳曼语支)
│   ├── C           ←  祖语言，如拉丁语之于罗曼语族
│   ├── C++         ←  C的"方言"进化，如法语之于拉丁语
│   ├── C#          ←  C++的"亲缘"但受Java影响，如西班牙语受阿拉伯语影响
│   ├── Java        ←  C语法壳但语义不同，如英语混合日耳曼+罗曼
│   ├── JavaScript  ←  Java之名但无亲缘，如日语借词非汉语系
│   ├── Go          ←  C简化版，如简体中文之于文言
│   └── Rust        ←  C语义但ML语法影响，如韩语混合汉字+固有词
│
├── ML/Haskell 分支 (≈ 印欧语系印度-伊朗语支)
│   └── (间接影响 Rust/Swift/Scala)
│
├── Smalltalk 分支 (≈ 汉藏语系)
│   └── (间接影响 Ruby/Objective-C/Swift)
│
└── 特殊语族
    ├── R           ←  S语言的进化，如冰岛语孤立保留古北欧特征
    ├── Python      ←  独立演进但吸收多方影响，如英语的混合性
    └── Swift       ←  混合语（Algol+Smalltalk+ML），如现代汉语混合语法
```

**教学法意义：** 学一门"亲缘语言"比学一门"异系语言"成本低——类比学法语后再学西班牙语 vs 学日语。Lang Master 利用亲缘关系推荐学习路径。

## 4. 元能力框架

### 4.1 七项核心元能力

| # | 元能力 | 定义 | 对应语言学概念 | 在 Top 10 中的体现 |
|---|---|---|---|---|
| 1 | **范式切换** | 能在 OOP/FP/过程式之间自由切换思维 | 语用层切换（code-switching） | Python(OOP↔FP) ↔ Rust ↔ Go |
| 2 | **抽象辨识** | 识别何为良好抽象 vs 过度抽象 | 形态层分析（morphological analysis） | C(零抽象) ↔ C++(模板) ↔ Rust(trait) |
| 3 | **语义直觉** | 理解"这段代码实际做什么"而非"这段代码怎么写" | 语义层理解 | 所有语言的核心挑战 |
| 4 | **类型思维** | 用类型系统而非注释来约束行为 | 句法-语义桥接 | Java(名义类型) ↔ Rust(代数类型) ↔ R(向量类型) |
| 5 | **错误语言** | 将错误处理视为语言设计的一部分 | 语用层-语用学 | Go(explicit) ↔ Rust(Result) ↔ JS(try/catch) |
| 6 | **并发心智** | 选择合适的并发模型而非默认选择 | 语用层-语用学 | Go(goroutine) ↔ Rust(async) ↔ JS(Promise) |
| 7 | **元编程意识** | 知道何时用代码生成代码 | 形态层-派生 | C++(模板) ↔ Rust(宏) ↔ Python(装饰器) ↔ JS(Proxy) |

### 4.2 元能力与语言覆盖率矩阵

| 元能力 | 最佳训练语言 | 辅助训练语言 | 训练模式 |
|---|---|---|---|
| 范式切换 | Python, JavaScript | Rust, Go | 同一问题用3种范式实现 |
| 抽象辨识 | C → C++ | Rust, Swift | 从零抽象渐进到高级抽象 |
| 语义直觉 | C(指针), Rust(所有权) | Java(引用), JS(闭包) | 语义对比挑战 |
| 类型思维 | Rust, Haskell思想 | Java, Swift, C# | 类型系统族谱探索 |
| 错误语言 | Rust, Go | Python, JS | 同一错误场景的5种处理方式 |
| 并发心智 | Go, Rust | JavaScript, Java | 并发模型对比练习 |
| 元编程意识 | C++, Rust, Python | JavaScript, R | 代码生成代码的三种流派 |

### 5.2 核心交互：概念级打字 → 概念级编程

Typing Master 是「看到单词 → 打出字符」。
Lang Master 是「看到概念 → 用指定语言表达」。

示例：
```
概念提示：  "对一个集合进行过滤并映射"
─────────────────────────────────────
Python:    [f(x) for x in items if p(x)]
JavaScript: items.filter(p).map(f)
Rust:      items.iter().filter(p).map(f)
Go:        // 没有内置filter/map，需要手写循环
C:         // 纯手动for循环+函数指针
Java:      items.stream().filter(p).map(f).collect()
```

**关键洞察：** 同一概念在不同语言中有不同的"表达密度"——这就是编程语言的 Sapir-Whorf 效应。

### 5.3 虚拟键盘 → 范式视图

Typing Master 的虚拟键盘显示"下一个要按的键"。
Lang Master 的范式视图显示"当前概念在该范式下的表达方式"。

```
[过程式视图]          [OOP视图]            [函数式视图]
for i in range(n):    class Counter:       range(n).map(f)
    result = f(i)       def next(self):        .filter(p)
                       return f(self.i++)
```
