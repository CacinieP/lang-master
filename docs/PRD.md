# Lang Master — 产品需求文档 (PRD)

> 版本：v0.1 | 日期：2026-05-26 | 状态：调研期设计稿

---

## 1. 产品概述

### 1.1 产品定位

Lang Master 是**编程语言的语言学**——不是另一门编程教程，而是帮助学习者从自然语言学和编程哲学的视角理解编程语言的结构、思维和迁移规律。

**一句话描述：** 看见同一概念在10种语言中的表达差异，理解每种差异背后的思维模式，掌握超越任何单一语言的7项元能力。

### 1.2 核心差异化

| 传统编程教育 | Lang Master |
|---|---|
| 按语言分章节："Python入门→Java入门→Rust入门" | 按概念分章节："变量在10种语言→函数在10种语言→并发在10种语言" |
| 学语法（Syntax） | 学五层结构（Lexical→Construction→Syntax→Semantics→Pragmatics） |
| 单语言锁定（"我是Python开发者"） | 元能力解锁（"我能用任何语言的思维模式思考问题"） |
| 模式匹配通过（记住 `map()` 的写法） | 语义验证通过（理解 `map()` 的行为并能在3种语言中辨识等价构造） |
| 不标注跨语言陷阱 | 显式标注负迁移陷阱（Java null → Rust Option） |
| 一门语言是"主语言" | 10种语言平等——没有特权语言 |

### 1.3 产品形态

- **Phase 1**: Web 应用（React + TypeScript + Vite），浏览器直接访问
- **Phase 2**: Electron 桌面端打包（可选，按需发布）
- **Phase 3**: 移动端适配（响应式设计，触屏交互优化）

## 2. 目标用户

### 2.1 三类用户画像

| 用户类型 | 代号 | 描述 | 入口语言 | 核心需求 | 预期时长 |
|---|---|---|---|---|---|
| **绝对初学者** | Zero | 无编程经验，想学编程但不知道选哪门语言 | Python（推荐路径B起点） | 建立编程直觉，不被单一语言锁定 | 3-6个月 |
| **语言切换者** | Switcher | 已精通1门语言，需要学第2/3门（如 Java→Go） | 已知语言（起点） | 利用正迁移加速，避开负迁移陷阱 | 1-3个月 |
| **多语言追求者** | Polyglot | 想精通多门语言，追求编程思维的广度 | 任意（全路径开放） | 范式切换速度、元编程意识、并发心智 | 6-12个月 |

### 2.2 用户画像详述

#### Zero（绝对初学者）
- **背景：** 大学生、转行者、好奇心驱动的自学者
- **痛点：** "我该学Python还是Java？"、"为什么同一个概念在不同语言中名字不同？"、"学了一门语言后，看其他语言的代码像看天书"
- **目标：** 建立"编程语言是语言"的直觉——理解编程语言有词法、句法、语义、语用，就像自然语言一样
- **路径：** 推荐路径B（Python→Java→C#→JS→Swift→Go→C++→Rust→C→R），但允许自由探索

#### Switcher（语言切换者）
- **背景：** 有2-5年经验的开发者，因工作/兴趣需要学新语言
- **痛点：** "我写Java的思维惯性太强，总是用Java风格写Go代码"、"Rust的所有权为什么这么难？"、"不同语言的错误处理方式完全不同"
- **目标：** 快速理解目标语言与源语言的差异，利用正迁移、规避负迁移
- **路径：** 从已知语言起步，Lang Master 自动标注与已知语言的正迁移/负迁移

#### Polyglot（多语言追求者）
- **背景：** 有5+年经验的高级开发者，追求编程思维的广度
- **痛点：** "我能在Python和Java之间切换，但函数式思维始终不自然"、"并发模型太多，不知道选哪个"、"元编程在不同语言中的哲学完全不同"
- **目标：** 精通范式切换、理解类型系统谱系、掌握并发心智模型
- **路径：** 全路径开放，重点训练7项元能力

## 4. 功能需求

### 4.1 按 C1-C7 约束组织

#### C1: 范式优先，语言次之

| FR-C1-1 | 概念导航树按概念域组织：变量→表达式→函数→类型→类/trait→并发→元编程 |
|---|---|
| FR-C1-2 | 语言选择不作为导航节点，而是作为概念内的"视图切换" |
| FR-C1-3 | 首页展示概念全景图而非语言列表 |
| FR-C1-4 | "语言差异速览"功能：从已知语言到目标语言的差异清单（专为Switcher） |

#### C2: c+1 原则

| FR-C2-1 | 推荐引擎：基于已掌握概念+依赖图，推荐恰好多1个新概念的练习 |
|---|---|
| FR-C2-2 | 概念密度标注：每个练习标注"新概念数"和"已知概念数" |
| FR-C2-3 | 难度分级 P1-P5 对应概念密度：P1=1新概念, P2=2已知+1新, P3=3已知+1新, P4=4已知+1新, P5=5已知+1新 |
| FR-C2-4 | 挑战模式只增加时间压力，不增加概念密度 |

#### C3: 负迁移必须显式标注

| FR-C3-1 | TrapAdvisor 组件：用户设置已知语言后，自动标注所有负迁移陷阱 |
|---|---|
| FR-C3-2 | 切换语言时弹出陷阱提示卡片（"从Java到Rust的陷阱：null→Option"） |
| FR-C3-3 | 概念节点包含 `traps` 字段，列出从每门语言的负迁移 |
| FR-C3-4 | 正迁移路径标注：从已知语言到目标语言的加速学习路径 |

#### C4: 三语言最低呈现

| FR-C4-1 | 概念页面至少展示3种语言的表达方式 |
|---|---|
| FR-C4-2 | 范式视图3列布局：过程式列 / OOP列 / FP列 |
| FR-C4-3 | 语言顺序随机（无Python优先） |
| FR-C4-4 | 允许展开查看更多语言（最多10种） |

#### C5: 五层始终可见

| FR-C5-1 | 左侧固定五层进度条：词法→构造→句法→语义→语用 |
|---|---|
| FR-C5-2 | 当前概念高亮所在层级 |
| FR-C5-3 | 每层独立追踪掌握率（而非笼统"已学X个概念"） |
| FR-C5-4 | 五层进度条可展开为详细视图，显示该层所有概念及状态 |

#### C6: 禁止纯模式匹配通过

| FR-C6-1 | 每个练习分两步：(1)代码表达 (2)语义验证 |
|---|---|
| FR-C6-2 | 语义验证题型：解释行为、选择正确输出、辨识等价表达式 |
| FR-C6-3 | 精通模式要求三步全对：代码+语义+等价辨识 |
| FR-C6-4 | 混合校验：简单概念用静态对比，复杂概念用沙箱执行验证输出 |
| FR-C6-5 | 沙箱执行架构：Web Worker 隔离，运行用户代码片段，验证真实输出 |

#### C7: 无特权语言

| FR-C7-1 | 概念展示时语言顺序随机排列 |
|---|---|
| FR-C7-2 | 数据结构中所有语言字段对称设计，无 `primary` 标记 |
| FR-C7-3 | 不使用"Python写X，其他语言也写X"的表述，而是"10种语言各有表达X的方式" |
| FR-C7-4 | 语言选择不影响概念导航路径 |

### 4.2 核心功能模块

| 模块 | 功能 | 描述 |
|---|---|---|
| **ConceptEngine** | 概念引擎 | 核心游戏逻辑——概念选择、练习生成、校验、评分 |
| **ParadigmView** | 范式视图 | 3列范式切换视图，展示同一概念在不同范式下的表达 |
| **MasteryTracker** | 精通追踪 | 概念掌握率追踪、元能力评分、五层进度 |
| **LayerNavigator** | 层级导航 | 五层进度条和层级详情视图 |
| **TrapAdvisor** | 陷阱顾问 | 负迁移标注、陷阱卡片弹出、正迁移路径推荐 |
| **SemanticValidator** | 语义验证 | 静态对比+沙箱执行的混合校验系统 |
| **RecommendationEngine** | 推荐引擎 | c+1推荐、学习路径规划、薄弱领域提示 |
| **CoursePackageManager** | 课程包管理 | 导入/导出/模板下载 |

### 4.3 三种学习模式

| 模式 | 入口 | 核心交互 | 校验方式 | 通过条件 | 数据记录 |
|---|---|---|---|---|---|
| **探索 (Explore)** | 概念导航树自由点击 | 浏览任意概念+任意语言的表达 | 代码表达（可选） | 无通过条件，自由浏览 | 浏览记录、停留时间、感兴趣的标记 |
| **挑战 (Challenge)** | 选择概念域+时间限制 | 限时内用指定范式解决问题 | 代码表达+语义验证 | 时间内完成≥80%正确 | 范式切换时间、正确率、概念密度 |
| **精通 (Master)** | 选择概念链（如"所有权链"） | 零错误完成从P1到P5的概念序列 | 代码+语义+等价辨识 | 100%正确，每步必须通过 | 完整概念链状态、元能力加分 |

## 5. 非功能需求

### 5.1 性能

| NFR | 指标 |
|---|---|
| 概念数据加载 | 首屏加载≤2s（IndexedDB缓存后≤500ms） |
| 概念切换 | 概念页面切换≤300ms |
| 沙箱执行 | 代码片段执行+输出验证≤5s |
| 10语言渲染 | 同一概念10种语言同时展示≤1s |
| 推荐计算 | c+1推荐计算≤200ms |
| 离线可用 | 无网络连接时100%功能可用 |

### 5.2 可访问性

| NFR | 描述 |
|---|---|
| 键盘导航 | 所有交互支持键盘操作（Tab/Enter/Esc） |
| 屏幕阅读器 | 语义化HTML+ARIA标注 |
| 深色模式 | 默认深色主题，支持切换浅色 |
| 字体缩放 | 支持150%字体缩放不破坏布局 |
| 国际化 | UI语言：中文/英文/法文（独立于编程语言概念数据） |

### 5.3 数据可移植性

| NFR | 描述 |
|---|---|
| 导出格式 | JSON + CSV 双格式导出 |
| 数据所有权 | 用户数据完全本地存储，无云端依赖 |
| 跨设备迁移 | 导出全部数据→另一设备导入→完整恢复 |
| 数据清除 | 一键重置，但先提示导出 |

### 5.4 安全性

| NFR | 描述 |
|---|---|
| 沙箱隔离 | 用户代码在 Web Worker + iframe 双隔离中执行 |
| 无外部API | 不调用任何外部API，数据不离开本地 |
| XSS防护 | 代码展示区使用 `<code>` + 文本渲染，不注入HTML |
| CSP头 | 生产环境配置 Content-Security-Policy |

### 7.2 Lang Master 的独特定位

**Lang Master 不与任何竞品在同一维度上竞争：**

- Codecademy/Exercism 教你"怎么写代码"——Lang Master 教你"为什么这样写"
- LeetCode 教你"如何解题"——Lang Master 教你"不同语言如何用不同思维解题"
- Rosetta Code 提供了对比数据但没有教学法——Lang Master 用语言学框架组织对比并强制语义验证

**核心定位：** Lang Master 是编程教育的"元层"——不是教编程语言，而是教"理解编程语言"的能力。

## 8. 风险分析

### 8.1 技术风险

| 风险 | 影响 | 概率 | 缓解策略 |
|---|---|---|---|
| 概念数据量过大导致 IndexedDB 性能下降 | 加载时间>2s | 中 | 概念数据按层分 store，懒加载；localStorage 缓存热门概念 |
| 沙箱执行不安全（恶意代码逃逸） | 用户设备被攻击 | 低 | Web Worker + iframe 双隔离；CSP 策略；代码长度限制≤50行 |
| 10种语言的代码片段维护成本高 | 内容质量下降 | 高 | 课程包社区贡献机制；模板+验证器自动生成 |
| 推荐引擎计算复杂度高 | 推荐>200ms | 低 | 预计算依赖图，离线排序推荐队列 |

### 8.2 产品风险

| 风险 | 影响 | 概率 | 缓解策略 |
|---|---|---|---|
| 用户不理解"语言学框架"的价值 | 不愿使用 | 中 | 首页用通俗类比解释（"编程语言就像自然语言有语法、词汇、语义"） |
| 概念数据覆盖不完整 | 学习路径断裂 | 高 | 分阶段发布：Phase 1 只覆盖 Python/C/Java/Rust 4种语言的P1-P3概念 |
| Switcher 不愿标注已知语言 | TrapAdvisor 不生效 | 中 | 入口时自然引导："你最熟悉的语言是？"— 非强制 |
| 绝对初学者被10种语言同时展示吓到 | 认知过载 | 中 | C2 约束：初学者只看3种语言（C4最低），不强制展示全部10种 |

### 8.3 优先级排序

| 优先级 | 功能 | 理由 |
|---|---|---|
| **P0 (必须)** | ConceptEngine, ParadigmView, MasteryTracker, LayerNavigator | 核心交互循环 |
| **P0 (必须)** | CONCEPT_DATA 数据模型和4种语言的基础概念集 | 没有数据就没有产品 |
| **P1 (重要)** | TrapAdvisor, SemanticValidator(静态校验部分) | C3和C6约束的核心实现 |
| **P2 (期望)** | RecommendationEngine, Challenge/Master 模式 | 飞轮闭环的关键 |
| **P2 (期望)** | SemanticValidator(沙箱执行部分) | 深度语义验证 |
| **P3 (可选)** | CoursePackageManager, 元能力雷达图, 语言热力图 | 增值功能 |
| **P3 (可选)** | Electron 桌面端打包 | Phase 2，不影响核心产品 |
